# 自定义域名指南

## 为什么建议绑定自定义域名

平台提供的 `vercel.app` 地址适合预览和早期使用。长期对外展示时，自定义域名更稳定、更专业。

最重要的好处是：域名由你控制。未来即使从 Vercel 迁移到 Netlify、Cloudflare Pages 或其他平台，只需调整 DNS，老师、律所、合作方和文章中使用的访问地址可以保持不变。

## 绑定步骤

1. 在正规域名注册商购买域名。
2. 登录 Vercel Dashboard，进入对应项目。
3. 打开 Domains 设置，添加准备使用的域名。
4. 根据 Vercel 页面给出的实时提示，到域名注册商或 DNS 服务商配置记录。
5. 等待 DNS 传播和 Vercel 验证。
6. 确认域名可以访问，并检查 HTTPS 证书是否自动签发成功。
7. 将 Vercel 环境变量 `SITE_URL` 更新为最终域名，然后重新部署。

## 常见 DNS 记录

- **A 记录**：把根域名指向平台提示的目标地址。
- **CNAME 记录**：把 `www` 等子域名指向平台提供的主机名。

不要照抄旧教程中的固定 IP。Vercel 的推荐值可能随平台配置变化，应始终以项目 Domains 页面当时显示的记录为准。

## 根域名与 www

可以选择：

- `example.com` 作为主域名；
- `www.example.com` 作为主域名；
- 两者都添加，并将其中一个重定向到另一个。

对外只公布一个主地址，避免链接分散。`SITE_URL` 应使用主域名，例如：

```text
SITE_URL=https://example.com
BASE_PATH=/
```

## DNS 生效与 HTTPS

DNS 变更可能需要几分钟到更长时间传播。等待期间可能出现不同网络访问结果不一致的情况。

检查项目：

- Vercel 中域名状态已验证；
- HTTP 会跳转到 HTTPS；
- 浏览器证书有效；
- 首页与子页面均能访问；
- canonical URL 使用自定义域名；
- 图片、CSS 和内部链接正常。

## 迁移平台时

1. 先在新平台完成部署并使用临时域名验证。
2. 在新平台添加相同自定义域名。
3. 根据新平台提示修改 DNS。
4. 确认 HTTPS 与页面正常后，再移除旧平台上的域名绑定。
5. 保留旧部署数日，便于 DNS 传播期间回退。

迁移时不需要修改网站内容，只需把新平台的 `SITE_URL` 设置为同一个自定义域名。
